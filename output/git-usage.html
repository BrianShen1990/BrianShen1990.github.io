Branch Operation

Why we need branch, branch are used during development. From my point of view, there are two main ideas:

1. We don't want our changes influence the main product code. At the meantime, we also want to use make changes. In this situation, we could make branch and hold our modifications in this branch.

2. We want to customize product for some customer, like logos, brands. The changes are aimed at one customer, so it's better to have an individual branch.

  • Add branch

    Warning

    Github Desktop

    Add a branch branch_new (Actually, when you create it, it is based on master if you use Github Desktop):

    Add files to the branch and commit and push. (Notice that the Current Branch tag indicates which branch you are dealing with.):

    After this action, in branch master, we only have one file, while in branch branch_new, we have two files.

    Warning

    Git

    Add files:

    Warning

    CMD

    git checkout -b branch_new                      # create a new branch
git add .\File_branch_new.txt                   # add file
git commit -m "Add new file to branch—new"     # commit
git push origin branch_new                      # push to remote

  • Merge Branch

    Warning

    Github Desktop

    Now, lets' merge branch_new to master

    Also, don't forget to merge. This time,branch master also have two files.

    Warning

    Git

    Use this menu to switch to branch master.

    Then merge:

    Remember to push to Github.

    Warning

    CMD

    git checkout master     # switch to master
git merge branch_new    # merge branch_new to master
git push origin master  # push to remote

  • Delete a branch

    Warning

    CMD

    git checkout -b branch_del              # create local a new branch with code of master
git push origin branch_del              # push it to remote
git checkout master                     # switch to master
git branch -d branch_del                # delete local
git push origin --delete branch_del     # delete remote
